# 시대생 크루 스케줄러 - 프로젝트 계획서

> 동아리 일정 관리 및 자료 공유 플랫폼
> **개발 기간**: 2026.06.20 ~ 2026.08.31 (약 10주)

---

## 1. 프로젝트 개요

### 1.1 배경

대학 동아리에서는 일정 공유, 과제 관리, 자료 배부를 위해 카카오톡 그룹채팅을 주로 사용하지만, 다음과 같은 문제가 있습니다:

- **일정 분실**: 중요한 일정이 채팅 속에 묻힘
- **자료 분산**: 과제, PPT, 참고자료가 채팅에 흩어짐
- **알림 부재**: 다음 모임 일정을 놓치기 쉬움
- **권한 부재**: 누구나 자료를 올릴 수 있어 관리가 어려움

### 1.2 프로젝트명

**스케줄러  (scheduler)**

### 1.3 목표

동아리 회원들이 일정과 자료를 체계적으로 관리할 수 있는 플랫폼을 구축합니다.

---

## 2. 기능 요구사항

### 2.1 핵심 기능

| 우선순위 | 기능 | 설명 |
|:---:|------|------|
| P0 | 인증 시스템 | 회원가입, 로그인, JWT 토큰 |
| P0 | 크루 관리 | 크루 생성, 가입 (초대코드), 회원 목록 |
| P0 | 일정 관리 | CRUD, 팀별 캘린더 조회 |
| P1 | 자료 공유 | 자료 업로드, 태그 검색, 다운로드 카운트 |
| P1 | 댓글 | 자료에 댓글 작성/조회 |
| P2 | 알림 | 이메일/Slack 연동 (Phase 3) |
| P2 | 채팅 | 실시간 채팅 (Phase 3) |

### 2.2 비기능 요구사항

- **성능**: API 응답시간 500ms 이하
- **보안**: JWT 기반 인증, BCrypt 비밀번호 해시
- **확장성**: Docker 컨테이너화로 향후 확장 가능
- **운영**: Swagger API 문서 자동 생성

## 3. 기술 선정

### 3.1 기술 스택

| 구분 | 기술 | 선택 이유 |
|------|------|---------|
| **언어** | Java 17 | 기존 역량 활용, 안정성 |
| **프레임워크** | Spring Boot 3.2 | 기존 프로젝트 경험 |
| **DB** | PostgreSQL 15 | 학습 목적, 가벼운 운영 |
| **ORM** | Spring Data JPA | 개발 속도 향상 |
| **인증** | JWT (jjwt) | Stateless, 확장성 |
| **API Docs** | SpringDoc OpenAPI | Swagger UI 자동 생성 |
| **빌드** | Gradle (Kotlin DSL) | 모던, type-safe |

### 3.2 시스템 구성

```
[Client]
   │
   ▼  HTTPS + JWT
[Spring Boot API] ─── [PostgreSQL]
   │
   ├── Swagger UI (개발시)
   └── Docker (운영시)
```
